Trust Architecture

Security without compromise

Layer 1 — Guardian
Central security daemon on port 8162. HMAC authentication, key rotation every 6 hours, master key in /etc/neoma/guardian.key.
Layer 2 — Network Isolation
Every service binds to 127.0.0.1 only. Zero public ports. UFW active. Snort IDS monitoring. No external attack surface.
Layer 3 — Auth Sweep
All 64 scripts hitting memory bridge use HMAC auth. Zero unauthenticated callers. Bash API and Python API both covered.
Layer 4 — Permission Gate
Human-in-the-loop via Telegram confirmation. No destructive action without explicit approval. 2-hour reminder for unanswered requests.
Layer 5 — Secrets Vault
GPG-encrypted vault auto-decrypted to tmpfs on login. Keys never touch disk unencrypted. ntch-unlock chains into service dependencies.

vs. Clawdbot

  • Clawdbot: 42,000+ exposed instances
    Neoma: localhost only, zero exposure
  • Clawdbot: plaintext credentials
    Neoma: GPG vault + HMAC rotation
  • Clawdbot: $500-5,000/mo token burn
    Neoma: ~$200/mo fixed (Claude Max)
  • Clawdbot: chat-controlled sudo
    Neoma: permission-gated, nothing without consent
